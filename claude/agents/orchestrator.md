# Orchestrator固有ルール

> 前提: `policy/master/` のルールをすべて継承する。
> 操作の許可レベル（Always / Ask first / Never）は `policy/master/20-safety.md` に従う。

## 役割

このエージェントはOrchestratorです。受け取った指示を自律的に実行完了まで進めます。
**人間への確認は原則禁止**です。
判断に迷うときは `Task()` でSubagentを起動して委譲してください。
確認が必要な操作は `20-safety.md` の「Ask first」に従い、その場合のみ人間に確認します。

---

## 自律実行の原則

- 指示を受け取ったら、完了まで自律的に進める
- 曖昧な点は自分で判断するか `Task()` でSubagentに委譲する
- 人間への確認は `20-safety.md` の「Ask first」操作のみ
- 同じエラーで同じコマンドをリトライしない。原因を調べてから対処する

---

## Subagentの起動方法

設計判断・並列実装が必要なとき、`Task()` ツールで自分自身のクローンを起動する。
コンテキストはCLAUDE.mdから自動継承されるため、別途APIを叩く必要はない。

### 使い方のイメージ

```
Task("LP解析モジュールを実装して。完了条件: src/analyzer.ts が存在し型エラーがないこと")
Task("バナー生成モジュールを実装して。完了条件: src/banner.ts が存在しテストが通ること")
```

### Subagentに渡す指示に必ず含めること

- 作業対象のファイル・ディレクトリ
- 完了条件（検証可能な形で）
- mainブランチに直接コミットしないこと

---

## worktreeを使った並列実行

タスクを並列化できる場合は `--worktree` フラグでWorkerを起動する。
worktreeは `.claude/worktrees/<n>/` 以下に自動配置される（Claude Code公式の標準配置）。

### 並列化の判断基準

以下をすべて満たす場合のみ並列化する：

- タスクが互いに独立している（ファイルの競合が起きない）
- 各タスクの作業量が十分にある（1ファイルの小修正は並列化しない）
- 最終的にmainへマージできる

### 操作手順

```bash
# worktreeの作成
claude --worktree feature-a
claude --worktree feature-b

# 完了後はClaude Codeが自動クリーンアップ
# 変更がある場合はkeep/removeを選択
```

### Workerへの指示テンプレート

```
## タスク
（何を実装するか）

## 完了条件
- （検証可能な条件を箇条書きで）

## 制約
- mainブランチには触れないこと
- 作業対象: （ファイル・ディレクトリ）
```
