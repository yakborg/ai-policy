# 20 - 安全ルール

## 編集前の確認手順

1. **対象ファイルを必ず読む** - 編集前に現在の内容を確認する
2. **変更の影響範囲を把握する** - 何が変わるかを言語化してから実行する
3. **可逆性を確認する** - 元に戻せない操作は事前にユーザーへ確認を求める

---

## 操作の許可レベル

### Always（確認不要で実行してよい）

- ファイルの読み取り・検索（Read / Grep / Glob）
- 単一ファイルの型チェック・lint・フォーマット
- git status / git log / git diff などの参照系操作
- テスト実行（単一ファイル・単一テスト）

### Ask first（実行前にユーザーへ確認する）

- `git push`（リモートへの反映）
- `git reset --hard`（変更の破棄）
- ファイルの削除
- GitHub PR / Issue の作成・コメント
- 外部サービスへのリクエスト（Slack、メール等）
- CI/CD パイプラインの変更
- システムパッケージのインストール（`apt install` 等）
- フルビルド・E2Eテストの実行（実行時間が長く、CI消費やコスト影響があるため）
- `source ~/.zshrc`（シェル再読み込み。意図しない副作用の可能性）
- `kill -9` / `pkill` の広範囲実行（プロセスの強制停止）
- `chmod -R` / `chown -R` の未確認実行（権限変更の広範囲影響）

### Never（いかなる状況でも実行しない）

- `~/.ssh/` 以下の編集・削除
- `~/.gnupg/` 以下の編集・削除
- `~/.secrets/` 以下のファイルを git add
- シークレット・APIキーをファイルや応答に平文で書く
- `rm -rf` を引数の確認なしに実行
- `eval` に未検証の動的文字列を渡す
- `/mnt/` 以下の Windows ファイルを削除
- `.env` ファイルを git でトラッキング

---

## 絶対に編集しないファイル

以下のファイルはユーザーが手動管理しており、AIによる自動編集を禁止する。

| ファイル | 理由 |
|---|---|
| `~/.zshrc` | シェル起動設定。変更するとシェルが起動しなくなる可能性がある |
| `~/.bashrc` / `~/.bash_profile` | 同上 |
| `~/.profile` | ログインシェル設定 |
| `~/.config/mise/config.toml` | ランタイムバージョン管理。誤変更で環境が壊れる |
| `~/.config/sheldon/plugins.toml` | プラグイン管理。変更後 `sheldon lock` が必要 |
| `~/.gitconfig` | Git のグローバル設定 |
| `~/.ssh/` 以下すべて | 認証情報。絶対に編集・削除しない |
| `~/.gnupg/` 以下すべて | GPG 鍵。絶対に編集・削除しない |

---

## ツール追加時のルール

新しいツールや設定を追加する場合は以下の順序で判断する。

```
1. mise で管理できるか？  → mise install <tool>
2. sheldon で管理できるか？ → plugins.toml に追記
3. npm/pip/deno でグローバルインストールできるか？
4. それ以外 → ~/.local/bin に手動配置
5. apt は最終手段（システム全体に影響するため）
```

## 不明瞭な指示への対応

- 指示の意図が不明な場合は**実行前に確認する**
- 「とりあえずやってみる」で破壊的操作を行わない
- エラーが出た場合は原因を調べてから対処する（同じコマンドをリトライしない）
