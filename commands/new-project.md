# 新規プロジェクト初期化

引数で指定されたディレクトリ（省略時は現在のディレクトリ）に新規プロジェクトを初期化する。

## 手順

1. 対象ディレクトリを確認する
2. プロジェクト名・スタック・主要コマンドをユーザーに確認する
3. `CLAUDE.md` を以下のテンプレートで作成する
4. 作成した `CLAUDE.md` の内容を表示して確認を求める

## CLAUDE.md テンプレート

```markdown
@~/.claude/CLAUDE.md

# <プロジェクト名>

## 概要
<プロジェクトの目的・説明>

## スタック
<言語・フレームワーク・主要ライブラリ>

## 主要コマンド
- dev: `<開発サーバー起動コマンド>`
- build: `<ビルドコマンド>`
- test: `<テスト実行コマンド>`

## ディレクトリ構成
<主要ディレクトリとその役割>

## 固有ルール・注意点
<プロジェクト固有の制約・禁止事項>
```

## 注意事項

- `@~/.claude/CLAUDE.md` は必ず先頭に記載する（グローバルポリシーの継承）
- 既に `CLAUDE.md` が存在する場合は上書きせず内容を確認してから対応する
- README.md など不要なドキュメントは勝手に作成しない
